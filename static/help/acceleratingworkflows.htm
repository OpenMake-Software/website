Accelerating Workflows

 

Workflows can be accelerated by executing them in what is called Parallelized Mode.  When a workflow runs in Parallelized Mode, omsubmit sorts out what Workflow Activities can be executed simultaneously, therefore speeding up the overall process.  In the same way that the Workload Manager, omsubmit, can manage “chained” Workflow Activities, it can also determine what Workflow Activities have no dependencies and can execute at the same time.  This helps improve the processing speed of the Workflow.  Workflow Activities that run in Parallelized Mode are executing on a single machine.  This is different from Distributed Workflow Processing where Workflow Activities are assigned to different Server Pools to execute.
