Web Client Workflows and JCL

In order to submit a Workflow from the Web Client, JCL must be written and added to the Workflow.  If a Build is Setup and executed via the ISPF panel, the corresponding JCL will be saved to a Workflow on the Knowledgebase Server and displayed via the Web Client.  This Workflow can then be re-executed via the Web Client without any additional coding of JCL.

 

To initiate a Workflow via the Web Client without using the ISPF panel, JCL must be added manually to the Workflow. Refer to the below section on defining your own custom JCL to execute OOMLBLDM and OOMLOM.
