IBM Rational

Activity Plug-in

Description

Update from ClearCase

 

This Workflow Activity takes updates from the ClearCase file repository and applies them to your workspace. This Workflow Activity usually occurs at the start of a Workflow before other activities are conducted. The Workflow Activity contains fields for Update Options and Snapshot-View-Path.  Other than these entries, all that needed is to ensure that the ClearCase tool is properly configured for your setup.

 

Create ClearQuest Build Record

This Workflow Activity creates a ClearQuest Build Record for the Workflow.  The Workflow Activity contains one field for the state.

ClearTool Exec - Command

 

This Workflow Activity executes a command with ClearTool Exec.  It sets the ClearCase view and then runs the command against that view.  The Workflow Activity contains fields for the ClearTool arguments, ClearCase View Tag, and command.

 

ClearTool Exec – Gen Meister Build Control File

 

This Workflow Activity calls Meister to generate a new Meister Build Control File with ClearTool Exec.  It sets the ClearCase view and then runs Meister against that view.  The Workflow Activity contains fields for the ClearTool arguments, ClearCase View Tag, Project, Search Path, and options.

 

ClearTool Exec – Incremental Meister Build

 

This Workflow Activity executes an OpenMake Meister incremental build using ClearTool Exec. Only files that are out of date will be re-built. It sets the ClearCase view and then runs Meister against that view.  The Workflow Activity contains fields for the ClearTool arguments, ClearCase View Tag, and flags. Note: You must execute a Meister Build Control File Workflow Activity before this step.

 

ClearTool Exec –Meister Incremental  Build with Clear Case Audit

 

This Workflow Activity executes an OpenMake Meister incremental build using ClearTool Exec while exposing all components used in the build, even when they are not managed by ClearCase.  It sets the ClearCase view and then runs Meister against that view.  The Workflow Activity contains fields for the ClearTool arguments, ClearCase View Tag, and Meister Arguments.  Note: You must execute a Meister Build Control File Workflow Activity before this step.

 

ClearTool Exec –Meister Clean All Build with ClearCase Audit

This Workflow Activity executes an OpenMake Meister re-build off all components using ClearTool Exec while exposing all components used in the build, even when they are not managed by ClearCase.  It sets the ClearCase view and then runs Meister against that view.  The Workflow Activity contains fields for the ClearTool arguments, ClearCase View Tag, and Meister Arguments. Note: You must execute a Meister Build Control File Workflow Activity before this step.

ClearCase Bill of Material Report

 

This Workflow Activity provides a ClearCase Bill of Materials report.  If the same Workflow has been run against the same source control module before, this Workflow Activity will also provide a ClearCase Differences report. This Workflow Activity should be one of the first activities defined in a Workflow because it takes a “snapshot” of the Build Directory for analysis and comparison with other iterations of a Workflow.  The Workflow Activity contains fields for many Workflow-specific parameters that are set by default to the corresponding Workflow Activity macros. The only parameter that will typically require attention is the field that indicates whether the Workflow is public or private.  Public Workflows need the “–lp” flag; private Workflows take no flag.

 

Execute Meister Incremental Build with ClearCase Audit

 

This Workflow Activity executes an OpenMake Meister incremental build while exposing all components used in the build, even when they are not managed by ClearCase.

When components are found in ClearCase, the version information of the source files from the ClearCase repository is reported.  The Workflow Activity contains fields for the Audit Report name, options, targets, and variable settings.  Note: You must execute a Meister Build Control File Workflow Activity before this step.
