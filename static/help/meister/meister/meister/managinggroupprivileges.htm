Managing Group Privileges

When a new Group is added, it is automatically granted full access privileges to all objects until the Administrator customizes the access privileges for the Group.  Given that full access is automatically granted, it is important to customize the access privileges of your new Group if you want to limit the Group's access privileges to Project Dependency Directories, Build Services, and Public Workflows.

 

 

Only the Administrator Group has the ability to edit Groups within the Groups Explorer. The access privilege within Groups allows you to manage objects at a very detailed level.  The access privileges of individual Project Dependency Directories, Build Services and Public Workflows can be assigned to each Group. These objects are managed at two levels, a "top level" and a "detail level".  Assigning privileges at the top level cascades the privileges to the detail level.  If more specific access privilege is needed, it can be assigned at the detail level.
