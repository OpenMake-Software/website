Remote Agents and Server Pools

 

Remote Agents are used to distribute the Workflow Activities across multiple machines and operating systems. Remote Agents are assigned to a Server Pool where the workload and server queue is managed. When the Workflow needs to execute across multiple machines and operating systems, each Workflow Activity can be assigned to execute on a particular Server Pool.  For example, if one activity needs to execute on Windows and another on Unix, then a Remote Agent can be installed on the appropriate machines, added to a Server Pool and the Workflow Activity is assigned to execute on that specific Server Pool.
