Initiating Workflows using JCL

 

To initiate builds using JCL, a job step must be defined for both OOMLBLDM and OOMLOM.   If you do not need to re-generate the Build Control file, you do not need to execute the OOMLBLDM job step.  However, this is not recommended as new Targets or changes to Targets may have occurred.  If you execute a build without re-generating the Build Control file, there is a risk of not building the load modules correctly.  Executing the OOMLBLDM job step ensures that an accurate build is done each time the build JCL is executed.
