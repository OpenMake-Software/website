Create and Edit Your Workflow

The Workflow Editor is the primary user interface that helps you create and edit your Workflows. Workflow Editor contains the following tabs: History/Trends, Monitor, Activities, Configure and Continuous Integration.

 

A Workflow is a collection of tasks steps executed as steps in a particular order.  A common Workflow may include the following steps: (1) check-out the most recent source code from a source code versioning tool, (2) execute a software build calling Meister or using manually coded build scripts (3) run a unit test,  and (4) send an email or instant message notification to announce that the Workflow has completed (5) deploy to a Server.   To setup your Workflow, the instructions in Table WM-1 will get you started.

 

Table WM-1 Naming a Workflow

Steps

Actions

1

Open the Workflow Explorer

2

Right Mouse Click on "Private Workflow" and Select "New"

3

Enter the name Sample Workflow in the New Workflow pop-up window.

4

Click the Finish button.

 

Your new Workflow will appear on the Workflow Explorer screen under Private Workflows. When you select and edit your Workflow from the Workflow Explorer screen, an editor pane will open displaying the History/Trends tab with two sections, the Workflow History and Data sections.  Across the bottom of the Workflow Editor are the following tabs:  History/Trends, Monitor, Activities, Configure, and Continuous Integration.
