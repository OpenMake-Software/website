The "System Info-Windows" Workflow

 

Table WF-3 will walk you through the steps of executing a more complex Windows Workflow.

 

Note:  If you are running Windows Vista, this workflow will execute, but fail on a call to a program that is no longer supported under Windows Vista.

 

Table WF-3: "System Info-Windows" Workflow

Step

Action

1

From the OM Workflow perspective, go to the Workflow Explorer.  Expand  on the Public Workflow node and double click on  "System Info-Windows".

2

A default output directory must be defined so that if a build directory is not defined to the Workflow then the Workflow knows where to write the results.  If you have not set a default output directory, a dialog box will be displayed.  Enter c:\build.

3

Select the Activities tab to go to the Activities View.  Double click on the "System Info-Windows" Activity.  This Workflow executes Activities to get the system and environment variable information and log it to a report. The results will be written to the directory referred to in the Build Directory field, c:\mojo\config.

4

Select the Overview tab and click Execute.

5

You will see the progress of the Workflow as it executes in the bottom pane of your screen in the Workflow Monitor.

6

Verify that your Workflow executed by reviewing the resulting logs.  From the OM Reports perspective, go to Workflow Log Explorer.

7

Expand the nodes  System Info-Windows – Today – My Computer.  Select the log.

8

The log will display your machines System Information and Environment Settings.  In this log, notice that each of the Activities are listed with a hyperlink to the Activities logging information.   Select "Log Windows Environment Report" to view the Windows Environment settings.
