ERROR 132:

No build control files were generated. Verify that the dependency extensions match the Build Service Rules extensions since the -s parameter may cause a mismatch.    

Description:

The dependency extension in the Target does not match any of the Build Services Rule extensions causing the dependency to be skipped, thus not creating the build control file.  This usually occurs when the Rules have changed and that the TGT has not need update accordingly.

 

Resolution:

Update the Target dependencies to match the Build Service Rules or Vic versa.
