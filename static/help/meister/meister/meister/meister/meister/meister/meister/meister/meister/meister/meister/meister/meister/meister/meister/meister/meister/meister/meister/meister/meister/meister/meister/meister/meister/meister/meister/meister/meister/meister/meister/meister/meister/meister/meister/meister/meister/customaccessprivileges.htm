Custom Access Privileges

For Build Services, Dependency Directories and Public Workflows, the following access privileges are available:

 

Read - Groups can view but cannot update the object.  If "Read" access is granted,  the Group will have the ability to see all of the objects, but will not have the ability to change the Objects.  This level of access is automatically defined for the default User Group.

 

Write - Groups can both view and update an object. "Read" access must also be granted for the Group.

 

Delete - Groups can delete an object. "Read" access must also be granted for the Group.

 

Execute - (Public Workflows only) Groups can execute a Public Workflow.  "Read" access must also be defined for the Group.

 

Add -  Groups can add new objects.   It is recommended that, if a Group has "Add" privileges, they also be provided "Write" privileges at the highest level.  This combined high-level access allows the Group to update objects they create.

 

None Selected - Users will not be able to view, update, delete, add or execute the objects.
