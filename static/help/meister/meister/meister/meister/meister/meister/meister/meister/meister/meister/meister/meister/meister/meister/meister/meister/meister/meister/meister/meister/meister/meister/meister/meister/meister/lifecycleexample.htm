Life Cycle Example

 

Tutorial Help:  View an on-line demo of how to create a Dependency Directory for a Life Cycle Model.  

For an example of a Life Cycle based Dependency Directory, view the "Metal Works" Project from the Project Explorer found under the OM Meister Perspective.  You will notice that there are four Dependency Directories defined for the "Metal Works" Project: DEV, INTEGRATION, QA and RELEASE.  This organization means that there are FOUR different types of builds that can occur based upon the four different Dependency Directory List.  A build can be performed for a Development Build, a QA Build, an Integration Build or a Release Build.  The difference between these builds is the location of the code that is found during the build.
