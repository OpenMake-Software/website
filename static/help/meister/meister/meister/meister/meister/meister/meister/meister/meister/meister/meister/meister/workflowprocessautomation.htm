Workflow Process Automation

Workflow automation can be used to execute any command line script in a particular order with relationships defined for each step in the Workflow.   Users enter the raw data (input) into Workflows and Workflow Activities.  The central Knowledgebase Server, which manages specific Workflow details, is updated when users save changes to their workflows and workflow activities.  Build Services and Workflow Services are also managed by the Knowledgebase Server and are called by the Workflow. Updating a Workflow also creates a metadata file (output) that organizes the activities associated with Workflows.  Once the activity metadata has been entered, a Command Line program called omsubmit performs the heavy lifting of managing your Workflow. omsubmit performs relationship analysis against your various Workflow Activities and executes a chain of Workflow Activities based on the success or failure of each step.        In addition, a command line program called omcommandline.jar initiates the work to omsubmit.
