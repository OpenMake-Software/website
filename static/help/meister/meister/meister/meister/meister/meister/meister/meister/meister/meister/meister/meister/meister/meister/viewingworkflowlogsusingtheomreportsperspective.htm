Viewing Workflow Logs using the OM Reports Perspective

 

To view your Workflow logs from the Eclipse Rich Client Platform, open the Workflow Logs Explorer from the OM Reports Perspective. Your previously saved Workflows will appear—when these are expanded, the corresponding log folders will also appear based on date. Clicking any log in the report folders invokes the Report Browser window on the right side of the screen. The Report Browser displays two log information fields. The first field, located on the left side of Report Browser, contains summary information, including the name of the Workflow (Build Job Name), the name of the user (Owner), the build machine (Machine), and the execution date and time (Date/Time). Also listed in this field are the individual activities with execution details. The second field, located on the right side of Report Browser, contains content information, including specific directories, configurations, running commands, and warnings, if applicable.

 

Note: If you intend to view Workflow Logs on AIX, HP-UNIX, or Sun Solaris, you must have Mozilla or Netscape installed. In addition, the install location of Mozilla or Netscape must be listed in the PATH environment variable.
