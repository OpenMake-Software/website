Executing Workflows on a UNIX Platform

 

If you are executing Workflows in UNIX, omsubmit uses the Bourne Shell environment to execute the back-end Workflow process on the UNIX operating system.  For this reason, the Bourne Shell environment for the integrated development environment under which the process is running must be set according to Bourne Shell syntax. If you have trouble executing activities on UNIX, log on to the UNIX machine using the integrated development environment that will execute the activities. Switch to the Bourne Shell, by typing sh.  Execute a Command Line build by running omsubmit from your Build Directory to determine if all of the needed environment variables have been set correctly.
