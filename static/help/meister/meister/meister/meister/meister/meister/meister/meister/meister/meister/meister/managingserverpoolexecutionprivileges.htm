Managing Server Pool Execution Privileges

A Group can be assigned “execute” privileges for a Server Pool.  By default, all Groups have access to all Server Pools.  Execute privileges enable the User to run a Workflow on a particular Server Pool. The security validation occurs at Workflow execution time.  This allows Users to define Workflow on a particular Server Pool, but restricts them from executing the Workflow on the Server Pool. This scenario is useful for Production level machines, where a development group may need to define the Workflow on behalf of a Production control group, but only Production control and perform the execute step.  Restricting the Execute access on a Server Pool does not restrict “Read” access to that Server Pool’s logs.  Restrictions on “Read” access to logs are handled at the Workflow privilege level.
