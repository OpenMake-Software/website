Naming a Project

Development project teams and Build Projects often have a one to one correspondence.  Therefore, the name of your development project is recommended to be the name of your Project. However, in some cases, you may need to define multiple Projects for your development project. If your development project has multiple release dates, you may want to define a separate Project for each release date, for example, "Metal Works 1.0" and "Metal Works 2.0". 

 

The only restriction on the Project Name is that it cannot have the following characters as part of the Name:

 

\    /    :    *    ?    "    <    >   |    &
