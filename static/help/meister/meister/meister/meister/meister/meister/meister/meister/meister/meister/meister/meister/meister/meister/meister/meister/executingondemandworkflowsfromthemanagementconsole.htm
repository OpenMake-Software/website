Executing on Demand Workflows from the Management Console

 

The Management Console provides you the ability to easily execute "Public" and “Private” Workflows and view all reports and logs associated to the Workflows.  Anytime you execute a Public or Private Workflow from the Management Control Console that has been defined to use a “local” machine, the Management Control Console defaults to the Knowledge Base Server as the "local" machine.  In other words if the Workflow is defined to run locally (defined as "My Computer" in the Workflow), then the Workflow will be executed on the Knowledge Base Server.  However, if the Workflow has been defined to use Server Pools with Remote Agents, the Workflow will execute using the Server Pool as configured.

 

To execute a Workflow from the Management Control Console, go to the "MyWorkflows” and choose “Latest Runs."  This will display a list of all Workflows showing the result of their latest run.  Select the desired Workflow and use the "Execute" button to run the Workflow.  You can view the execution of the Workflow by going to the “Workflow Monitor” menu option under “MyWorkflows.”
