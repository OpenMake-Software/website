Build Management

 

Meister for z/OS uses the Dynamic DevOps Suite central knowledge base for managing the creation of load modules.  All of the generic rules and scripts used to create load modules are managed via the OpenMake Dynamic DevOps Knowledgebase Server.  Users of Meister for z/OS generate build control information by creating a Target (TGT) file that contains the specific high-level information about that particular final target.  With the specific Target information, using the Dynamic DevOps  Knowledgebase, the compile JCL is generated on an as needed basis, determined by Meister for z/OS OOMLOM program.  Meister for z/OS's OOMLOM program uses the build control file to determine what dependencies need to be rebuilt.  As it finds dependencies that are out of date, it creates the appropriate compile JCL and submits the JCL to the internal card reader for execution.
