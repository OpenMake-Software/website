Using a Batch script to define “Global” Variables

 

A batch script (.bat or .sh) can be used to define environment variables at the “global” level.  The batch script should contain a list of the variables and their values.  This batch script can then be executed as the first step in the Workflow.  All subsequent Workflow Activities will inherit these globally set environment variables. To add scripts that may affect numerous environment variables or other configuration settings, click on the New Script button and enter the name of the script into the New Script pop-up window in the Script input line. The script will display in the configure environment field as Run.
