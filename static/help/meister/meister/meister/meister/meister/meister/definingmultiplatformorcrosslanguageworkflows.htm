Defining Multi-Platform or Cross Language Workflows

 

Workflows can execute Workflow Activities across multiple Server Pools simultaneously regardless of Operating System.  This capacity is used to support multi-platform Workflows where a single application uses multiple operating systems or a single application may need to be re-compiled to support multiple operating systems. 

 

Multi-platform Workflow are supported through Server Pools.    To define a multi-platform Workflow, you must install Remote Agents on each of the target platforms and assign the Remote Agents to a Server Pool using the Management Control Console.  All Server Pools will be displayed through the Workflow Explorer, regardless of Operating System, by name.  Workflow Activities for each of the Server Pools can be defined for each multi-platform machine and executed as a single process within the Workflow down to the task level. 

 

 

 To set-up distributed Workflow processing you simply configure your Workflow to use Server Pools for the entire Workflow or for each Workflow Activity.

 

Note:   If your distributed Workload requires chaining between two different machines, you must use a Nested Workflow
