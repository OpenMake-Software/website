Defining Your Program

 

Your program (a single load module) is referred to as a Target.  Projects are a collection of Targets. 

 

Targets are created in many different ways called Build Types.  The Target Table, available from the Setup Build ISPF panel or the Web Client, displays the specific Build Types with which the Target is associated.  All Targets displayed in the Target Table belong to the same Project and are displayed based on the Search Path. If you have multiple Search Paths, you may see different Targets on your Target list.  This variety occurs when different data sets in your Search Path return different versions of the Target files. 

 

Note: If no Targets are found for your Project, this may mean that no .TGT files were found in the Search Path you selected.  Check to make sure that your Project Search Path has been defined correctly.  If your TGT files are in your local working data set, your Search Path must have a "."(period). It is recommended that the "." be the first entry in your Search Path.
