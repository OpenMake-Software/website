Distributed Workload Processing

 

 

You can define particular Workflow Activities to be executed on specific Server Pools of Remote Agents when you set-up your Workflow.  This allows the individual Workflow Activities to be executed across multiple Remote Agents for improved processing speeds.   This is useful when you have multiple Workflow Activities that must execute under different operating systems or you would like to distribute the workload across multiple machines, virtual or physical.
