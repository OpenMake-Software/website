.NET and  "Project" Dependency Directories

 

For all Microsoft .NET Build Services (C#, C++ and Visual Basic), modifications to the Dependency Directories are necessary for managing builds that use Microsoft .NET Solution information.  All Dependency Directories associated with .NET Solutions must be marked to use Microsoft "Project" directories.  Table CP-6 will guide you through the process of defining your "Project" directory.

 

 

Table CP-6  Define a Directory as a "Project" Directory

Step

Action

1

From the main menu select Window-Open Perspective-OM Meister

2

Select the Project Explorer tab near the top of your screen.

3

Double Click on the Dependency Directory Name which you wish to modify.

4

You will be provided an Overview screen on the right side of the Project Explorer

5

Select the Directory to modify and select the Edit button on the right of your screen.

6

Select the Project Directory radio button and select OK.

 

You must be an Administrator or have write access to the Dependency Directory object to make this change.  This specification is needed because the Microsoft Solutions use a separate directory for each program.  Marking the Dependency Directory with the 'Project Directory' radio button  makes it possible to handle the Microsoft builds correctly.
