Creating Mainframe Targets

 

Developers set up builds by defining their load modules, referred to as Targets.  Targets are organized based on the Project and Search Path.   The Setup Build screen allows Developers to add Targets to the Project, based on a particular release level called a Search Path. The Search Path organizes Projects based on different stages of the application life cycle or different release levels.  Search Paths contain a list of data sets from which to locate the correct level of the source code. Targets are found based on the selected Search Path.  Datasets listed in the Search Path will be read and any Target files are located for the specific Target.  Each Project should contain at least one Target.  In most cases, each Project will contain many Targets.
