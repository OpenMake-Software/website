Synchronizing Knowledgebase Servers

 

If you want to be able to build the same projects in a test environment that exist in production, you may want to update the test server with changes made to prod. These changes are usually on-demand changes made to the live production server such as new or modified search paths. To synchronize the changes is to copy the changed files from the prod server to the test server.  Table MD-2 describes the files that need to be copied.

 

Table MD-2 Files to Copy

Folder (Relative to openmake.war/)

kb/jobs

kb/projects

kb/build types

kb/activity descriptors

scripts

The kb/groups and kb/users directory can be ignored since different users and groups can exist on the test server.

 

The Knowledgebase Server being updated will need to be restarted in order to recognize the changes.

 

Note:  The Workflows may need to be edited in order to change the Remote Agents defined to new Knowledgebase Server.
