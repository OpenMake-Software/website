Workflow Activity Chaining

 

Workflow Activities can be defined with dependent relationships.  For example, a source code check-out must complete before a software build is executed.  With “chained” Workflow Activities, you can define your life cycle logic to execute in a particular order.  There also may be cases where multiple activities can be executed at the same time.  This is called running your Workflow in Parallelized mode.  The process is managed by the Workload manager called omsubmit.
