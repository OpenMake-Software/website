Intermediate Directory

 

In order to easily reference final Targets after a build, it is often useful to save all intermediate Targets, separate from the final Targets, in a data set containing only intermediate Targets (called an "Intermediate Directory").  If the Build Type you have selected supports an Intermediate Directory, then you can specify the name of the Intermediate Dataset.  If the Build Type you have selected does not support the use of an Intermediate Directory, this field will not be used.  If the Build Type you have selected allows the use of Intermediate Directories but you do not name an Intermediate Directory, The "Final Directory" will be set as your default "Intermediate Directory".  The Final Directory is normally your Load directory based on you High Level Qualifier.  All references to an Intermediate Directory by the Build Rule will default to the Final Directory Name if the Intermediate Directory name is not specified.
