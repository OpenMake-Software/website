Changing the Status of  your  Workflow

 

You can update the status of a Workflow  using the Trend table accessible from the “Lastest Run” or “Previous Runs” Menu options found under “MyWorkflows” on the Management Control Console.  The default settings for Build Status are:

Failed – The Build Step of the Workflow produced Errors. This Status is automatically set when the Build completes with Errors.

Clean – The Build Step of the Workflow completed without Errors.  This Status is automatically set with the Build completes without Errors.

Testing – The Status of a Build can be set to Testing, indicating that this Build is at the Testing Phase.

Released- The Status of a Build can be set to Release, indicating that the Build is ready for deployment, or has been deployed.
