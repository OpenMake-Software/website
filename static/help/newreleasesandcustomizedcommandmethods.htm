New Releases and Customized Command Methods

 

At times it may be necessary to provide patch releases to the standard Methods.  These patches generally are needed to support compiler upgrades that require the Methods to be updated in order to take advantage of the compiler enhancements.

 

When a new release of the Methods is applied to your Knowledgebase Server, the previous versions of the Methods will be written over by the new version.  For this reason it is highly recommended that none of the supplied Methods be modified.  A safer method is to copy the standard Method to a new name, and modify the new Method.  Once you have your new Method modified, using the Web Client, perform a copy function of the old Build Type to a new name and then re-associate the Build Rules to you customized Method.  This approach preserves the standard Build Types, Build Rules and Command Methods and prevents your script from being written over during an upgrade.
