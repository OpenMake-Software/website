Default User Group

The default User Group has full access privileges to the Target Explorer, Workflow Explorer, and the Reports Explorer with read access to Build Services, Dependency Directories and Public Workflows. As long as a User is in the default User Group, they will always have full access privileges to all these Views, as well as read only access to the Administrative Views.  However, if a User is assigned to both the default User Group and a Custom Group that has specific Administrative privileges, the Administrative privileges will override the default User privileges. For example, a Group is created that has the ability to update Windows Build Services and a User is added to the Group. The User will then have write access to the Windows Build Services and read access to all of the other Administrator Views.
