Naming Projects

 

Development Projects and Build Projects should have a one to one correspondence.  Therefore, the name of your development project is recommended to be the name of your OpenMake Project. However, in some cases, you may need to define multiple Build Projects for your development project. If your development project has multiple release dates, you may want to define a separate OpenMake Project for each release date, for example, "Payroll 1.0" and "Payroll 2.0". 

 

The only restriction on the Project Name is that it cannot have the following characters as part of the Name:

\  /  :    *    ?    "   <     >   |   &
