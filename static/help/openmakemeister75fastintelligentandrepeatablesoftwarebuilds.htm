OpenMake Meister® 7.5 – fast, Intelligent and repeatable software builds

 

 

OpenMake Meister® delivers a model driven and accelerated build solution that can quickly adapt to changing development demands. Similar to the model driven ARA solutions, Meister gives you a framework for developing reusable models for defining the logic that drives your compile/link/archive process eliminating the need to maintain a one-off script for every build.  It accelerates builds, relieves congestion, stores binaries and gathers critical dependency mapping that can be used for release packaging.

 

OpenMake Meister includes a CI Server or can be called by any CI server of your choice

Meister's difference is in the way it delivers the full potential of the Build management process combined with Continuous Integration Workflow.  Meister integrates with build engines to create synchronized pre-commit, continuous and team builds for today's quickly changing development environment.  

 

Through Build Management and Automation, developers can maximize on feed back loops and accelerate Builds for multiple languages including C/C++ and Java. By addressing the complexities found in the Build process, Meister offers development organizations a simplified and standardized process for managing the application Build and Workflow across diverse languages and platforms.  Meister integrates with more IDEs, development tools, and ALM products than any of its competitors.  Setting up Meister to manage your Builds requires a minimum of time and effort, and provides an effective and expansive tool for automating your Build processes.  Meister’s features include:

 

•     A 100% complete solution offering both Build Management and a fee Continuous Integration Server.

•     Build coordination for cross platforms and languages including C, C++ and Java.

•     Build acceleration through multi-threaded and parallelized builds.

•     Reduced development cost through improved Build efficiency.

•     Pro-active Build corrections to eliminate “bad builds”.

•     Enhanced Continuous Integration Build support.

•     Designed for complex “compile and link” software Build processes.

•     Source Code Dependency management linking production binaries back to development origins, with or without a Source Code Management repository.

•     Easy to use Management Control Console for information sharing and Workflow execution.
