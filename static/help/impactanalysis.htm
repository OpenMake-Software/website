Impact Analysis

 

Impact Analysis is truly a luxury in the overall development process.  The difficulty with impact analysis is that it must be easy to implement and be easy to read.  Meister for z/OS handles impact analysis using a simplified method.  First, to build an impact analysis report, a flag is passed to the OOMLOM program.  When OOMLOM reads the Build Control file, it scans all dependencies, recording date, and time.  It then generates a report in XML format.  Secondly Meister for z/OS displays the XML report in two views, the Implosion View and the Explosion View.  The Implosion View shows all executables impacted by a source module.  The Explosion View shows all impact, starting with the load module, working down through the libraries and source code.
