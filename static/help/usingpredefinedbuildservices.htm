Using Pre-Defined Build Services

 

 

Build Services provide build best practices and build comprehension for multiple operating systems and languages.  Build Services clarify how a build is managed, providing a means for developers to comprehend how a build is processed from beginning to end.  Build Services are reusable, preventing developers from having to re-invent the build process each time they need to build a binary (.jar,.war,.dll, .o, etc).  Build Services allow developers and build administrators to write highly reusable build scripts, eliminating the redundancies and errors normally found in repetitive manual scripting. Build Services are organized based on Operating System and can be found under the OM Meister perspective using the Build Service Explorer.
