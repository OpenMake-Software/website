Naming a Project's Dependency Directories List

Dependency Directories are simply a list of Directories that can be named as a group.  Therefore, the name of your list of directories should be named according to the "Build Label", "Version View", or "Life Cycle State" for which they represent. The are two main types of Dependency Directories, one based upon a life cycle model and the other based upon a code branching model.  How you name the Dependency Directory depends on how you manage your version process.

 

The only restriction on the Dependency Directory Name is that it cannot have the following characters as part of the Name:

 

\    /    :    *    ?    "    <    >   |    &
