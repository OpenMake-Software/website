Managing Users with OM Admin

 

Managing Users and Groups is done through the OM Admin Perspective by going to the main menu and selecting Windows – Open Perspective – OM Admin.

 

You will use either the User Explorer or Group Explorer to view and assign Group privileges.  By default, there are two Groups pre-defined, an Administration Group and a User Group.  The User ID used to perform the installation is automatically added to the Administration Group. Any users connecting to that Knowledge Base Server after the installation are automatically assigned to the User Group. 

 

 

Note:  During installation, each person installs their own instance of the Knowledge Base Server, therefore can administer their local Knowledge Base.  To create a Community Developed Knowledge Base, each person must connect to the central Knowledge Base. This is done by going to the main menu and selecting Window-Preferences-Openmake-Machine and changing the Machine Host configuration to the Community Developed Knowledge Base configuration information.
