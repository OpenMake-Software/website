Adding Workflow Services to a Workflow

 

Several useful Workflow Services have been pre-created to allow for rapid “out of the box” implementation of common Workflow Activities.  You can view the Workflow Service Explorer tree to see what Services are available.

 

Once created, Workflow Services can be easily added to existing Workflows. Remember that Workflow Services are stateless, so they will automatically inherit the machine and environment properties of the “parent” Workflow.

 

The Workflow Service Activity plug-ins are dynamically updated as new Workflow Services are added. Following are the steps for adding an existing Workflow Service to a Workflow.

 

Table WM-16 Adding Workflow Service  to a Workflow

Steps

Actions

1

Within an existing Workflow, select the Activities tab.

2

Click the Add Activity button. A window appears with two fields: one for plug-ins (Activity Plug-in) and the other for activities (Activity Type).

3

Select the Workflow Service Plug-In to see what Services are available. Choose your desired Service and optionally provide a name.

4

Click OK. Your new Workflow Service will appear under the machine that you chose. 

5

Optionally, define the derived parameters for the Workflow  Service. Note: if left parameterized and undefined, users will be prompted for their values at execution time.
