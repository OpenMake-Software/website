Batch Build Logs 

 

When OOMLOM is executed in batch mode it will not report back to the Knowledgebase Server the location of the Build Log.  When OOMLOM is executed directly via the batch the output is sent to SDSF in the Held Output queue.  If you want Build Logs saved to the Knowledgebase Server, you can pass to OOMLOM output flags indicating how you wish the output to be logged.  Refer to the OOMLOM flags for specifying your output method.
