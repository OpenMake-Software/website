Using Batch Scripts to Set Workflow Activity Variables

 

A batch script (.bat or .sh) can be used to define environment variables at the Workflow Activity level.  The batch script should contain a list of the variables and their values.  This batch script can then be executed before the Workflow Activity.  If a Workflow Activity Level variable overrides the “Global” Workflow variable, the “Global” variable will be reset once the Workflow Activity is completed. To add scripts that may affect numerous environment variables or other configuration settings, click on the New Script button and enter the name of the script into the New Script pop-up window in the Script input line. The script will display in the configure environment field as “Run.”
