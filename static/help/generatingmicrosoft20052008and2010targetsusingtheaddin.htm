Generating Microsoft 2005, 2008 and 2010 Targets using the Add-In

The Microsoft Add-in automatically generates your Target based on the Microsoft Project file.  This is a useful process for supporting continuous integration where your builds running inside of the IDE must be synchronized with the build running outside the IDE on your continuous integration server. This synchronization resolves changes made to the source code that has impacted the build scripts running outside of the IDE such as the case with refactoring, adding libraries, or changing compile options. 

 

Note: Different versions of the Microsoft IDE have different specific requirements in using the Microsoft Add-in. Refer to your unique Microsoft IDE for specific details.
