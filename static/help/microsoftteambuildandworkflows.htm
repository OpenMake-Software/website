Microsoft Team Build and Workflows

 

In order for Team Build to execute Meister’s Build automation features, the Meister Build must be delivered through a Meister Workflow.  To do this,  Microsoft Team Build must be configured to recognize a Meister Workflow. This configuration is done using the Team Build – Build Definition files. 

 

Execution of a Workflow can be done through an existing Team Build  - Build Definition file or through a new Build Definition file.  A Build Definition file is a Microsoft MSBuild Project File that contains the information to enable the Team Build Server to execute custom builds, stored in XML format.
