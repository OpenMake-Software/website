Input and Output

 

Meister for z/OS is organized into two main input and output components:

 

The Web Client/Native ISPF Panels:

Input: Data entered by the User

Output:  The Knowledgebase Server data and Target Definition Files

 

The Command Line Programs

OOMLBLDM :

Input: The Knowledgebase Server Data and Target Definition Files

Output: Build Control File

    

OOMLOM :

Input:  Build Control File

Output: Build Results, Log and Impact Analysis
