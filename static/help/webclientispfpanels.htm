Web Client & ISPF Panels

 

Meister for z/OS features two different interface options: the Web Client and the ISPF Panels. All administrative features of Meister for z/OS, such as defining Groups, Users, Projects and Build Types must be done through the Web Client.  The ISPF Panels provides Developers a native z/OS interface for creating Targets and submitting builds.

 

Impact Analysis reports and stored Workflow logs are viewed via the Web Client.  Reporting features are not available through the ISPF Panels as the reports are stored in HTML format and displayed in a graphical manner.  For this reason, reports cannot be viewed via the ISPF Panels.

 

Both the Web Client and the ISPF Panels displays and collects data related to Builds and both connect to the Knowledgebase Server to read and write build information.
