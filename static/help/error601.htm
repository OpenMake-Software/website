ERROR 60:

          Could not open Build Control File

 

Description:

The -f parameter was used with bldmake indicating that it should take the Project and Dependency Directory information from an existing Build Control File, however no Build Control File (*.mak) was found.

Resolution:

Make sure that a Build Control File (*.mak) exists in the directory where bldmake is running. If no Build Control File exists, remove the -f flag and pass bldmake both the Project and Dependency Directory parameters.
