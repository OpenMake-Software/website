Post-Build Activities in Visual Studio .NET Projects

 

If you are using the Meister build engine and have post-build commands in your Visual Studio projects, the post-build commands will be ignored. This allows the Meister build engine to build in different environments without relying on user or machine configuration. It is recommended that you add any desired pre-build or post-build commands to the Meister Workflow as Workflow Activities. While the Workflows may be machine-specific, the build engine, which relies on the project files, is not.
